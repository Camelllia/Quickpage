# PRD — 퀵페이지 (QuickPage)

**버전**: 1.0 (Portfolio MVP)  
**작성 목적**: 기획자 포트폴리오 · 가설 검증용 프로토타입  
**한 줄 정의**: 개발자·디자인 툴 없이 10분 안에 이벤트·모집 랜딩 페이지를 만들고 배포하는 초간편 빌더

---

## 1. 배경 & 문제

### 1.1 왜 이 제품인가

이벤트 안내, 모집 공고, 세일 프로모션, 공지 페이지는 **수명이 짧고 반복 제작**이 많다. 그런데 실무에서는:

- 외주·개발 요청 → **며칠~수주** 소요
- 미리캔버스·피그마 → 디자인은 되지만 **배포·트래킹·수정**이 분리됨
- 노션·구글사이트 → 빠르지만 **랜딩·전환(CTA)에 최적화되지 않음**

→ **「빨리 만들고, 바로 공유하고, 효과를 보고, 다음에 또 빠르게」** 할 도구가 필요하다.

### 1.2 핵심 가설

| ID | 가설 | 검증 방법 |
|----|------|-----------|
| H1 | 비개발 직군도 템플릿만으로 10분 내 페이지를 배포할 수 있다 | 편집기 Golden Path 완성 + 사용성 테스트 |
| H2 | UTM 트래킹·방문 로그가 마케터에게 가치 있다 | 트래킹 링크 생성·대시보드 제공 |
| H3 | 무료→유료 전환은 워터마크 제거·분석·도메인으로 가능하다 | 요금제 설계 + (향후) 결제 연동 |

## 3. 사용자 시나리오

### 시나리오 A — 봄맞이 오프라인 이벤트 (마케터)

1. `/templates`에서 「이벤트 클래식」 선택
2. 배너 업로드, 일시·장소·혜택·CTA 입력
3. 레이아웃·글꼴·색상 조정, MO 미리보기 확인
4. 로그인 후 **배포하기** → `/p/{id}` 링크 확보
5. 트래킹 링크로 `utm_source=instagram` 생성 후 SNS 게시
6. 대시보드 **방문 로그**에서 유입 확인

### 시나리오 B — 강사 모집 공고 (학원 운영)

1. 「모집 공고」 템플릿 선택
2. 혜택 목록·신청 버튼(구글폼 링크) 설정
3. 배포 후 카카오 채널에 링크 공유
4. 필요 시 `/create?edit={id}`로 문구만 수정·재배포

### 시나리오 C — 비로그인 사용자 (전환 유도)

1. 템플릿으로 편집·미리보기까지 진행
2. **배포하기** 클릭 → 로그인 유도 모달 (알럿 아님)
3. 로그인 후 편집 내용 유지·배포 완료

## 4. 제품 범위 (MVP)

### 4.1 In Scope — ✅ 구현 완료 (프로토타입)

| 영역 | 기능 |
|------|------|
| 템플릿 | 6종 프리셋 (이벤트, 모집, 세일, 공지, 오픈, 워크샵) |
| 편집기 | 섹션 토글, 배너(크기·포커스·fit), 혜택 목록, CTA |
| 레이아웃 | 6종 (classic, hero-overlay, split, minimal, magazine, card) |
| 디자인 | 테마 프리셋, 강조/배경 색, 글꼴 7종, 정렬·제목·CTA 스타일 |
| 미리보기 | MO/PC 토글, 실시간 렌더링 |
| 배포 | 로그인 필수(Supabase), `/p/{id}` 공개 URL |
| 관리 | 대시보드, 폴더, 이름 변경, 수정·삭제 |
| 트래킹 | UTM 파라미터 수집, 방문 분석 대시보드, 트래킹 링크 생성 |
| 추출 | HTML·PNG 다운로드 |
| 인증 | 회원가입·로그인, 배포 시 로그인 유도 모달 |

### 4.2 Out of Scope — 📋 설계만 (포트폴리오·Phase 2)

| 기능 | 제외 이유 |
|------|-----------|
| 결제·구독 | MVP는 가설 검증 우선, 수익화는 로드맵 문서화 |
| 플랜별 하드 리밋 | 요금제 UI는 있으나 강제 로직은 Phase 2 |
| 커스텀 도메인 | Business 플랜 약속, 인프라·DNS 연동 비용 큼 |
| 팀 계정·권한 | B2B 확장 단계 |
| 내장 신청 폼 | 1단계는 CTA→외부 링크, 2단계에서 Supabase 폼 |
| 페이지 자동 만료 | 설계 완료, 구현은 우선순위 조정 |
| OG 메타 자동 생성 | 로드맵 포함, 공유 최적화 Phase 2 |
| 정적 HTML CDN 배포 | 장기 차별점, 아키텍처 문서에 명시 |

### 4.3 의사결정 기록 (ADR 요약)

| 결정 | 대안 | 선택 이유 |
|------|------|-----------|
| JSON + 단일 렌더러 | 페이지마다 커스텀 HTML | 유지보수 0, 템플릿 확장 용이 |
| 배포 시 로그인 필수 | 익명 배포 허용 | 페이지 소유·대시보드 일관성 |
| 트래킹을 대시보드로 이동 | 공개 페이지에 로그 노출 | 운영자 UX·프라이버시 |
| 글꼴 드롭다운 | 전체 그리드 노출 | 편집 패널 공간·인지 부하 감소 |
| 로그인 모달 | `alert()` | 전환율·브랜드 톤 유지 |
