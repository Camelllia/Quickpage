| 대안 | 강점 | 약점 | 퀵페이지 차별 |
|------|------|------|----------------|
| **미리캔버스** | 디자인 자유도, 템플릿 풍부 | 배포·호스팅·UTM 분리, 러닝커브 | 폼 중심·즉시 `/p/url` 배포 |

## 2×2 포지셔닝 (개념)

```
        높은 커스터마이즈
              │
   Webflow     │     외주
              │
  ────────────┼────────────  빠른 제작
              │
  미리캔버스   │   ★ 퀵페이지
              │
        낮은 커스터마이즈
```

퀵페이지는 **빠른 제작 + 랜딩 목적 최적화** 사분면.

---

## SWOT (요약)

| | |
|---|---|
| **S** | 명확한 니치, 동작 프로토타입, 트래킹·대시보드 일체 |
| **W** | 브랜드 인지 없음, 결제·도메인 미구현, 템플릿 수 제한 |
| **O** | 소상공인·교육·이벤트 시장, 숏폼→랜딩 유입 증가 |
| **T** | 범용 툴의 랜딩 템플릿 강화, 무료 호스팅 대안 |

---

## 전략적 선택

1. **넓게 가지 않기** — 웹사이트 빌더 X, 「단기 이벤트 페이지」 O  
2. **속도·배포·트래킹** 삼박자를 Golden Path에 몰기  
3. Pro는 **워터마크 제거 + 분석**으로 마케터 지갑 공략  
4. Business는 **팀·도메인**으로 에이전시 (후순위)
