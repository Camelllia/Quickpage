# 3분 데모 스크립트

> 면접·포트폴리오 리뷰·발표용. **약 2분 30초 ~ 3분 30초** 기준.

---

## 사전 준비

- [ ] Supabase 연동·로그인 완료
- [ ] 브라우저 시크릿 창 1개 (비로그인 시나리오용, 선택)
- [ ] 샘플 배너 이미지 1장

---

## 스크립트

### 0:00 — 문제 제시 (20초)

> 「이벤트나 모집 페이지는 자주·빨리 만들어야 하는데, 외주는 느리고 미리캔버스는 배포와 트래킹이 분리돼요. 퀵페이지는 **10분 안에 만들고 바로 공유**하는 데 집중한 기획·프로토타입입니다.」

**화면**: `/` 랜딩 히어로

---

### 0:20 — 템플릿 선택 (25초)

> 「마케터가 가장 많이 쓸 이벤트 템플릿부터 보여드릴게요.」

**동작**:
1. `/templates` 이동
2. 「이벤트 클래식」 또는 「모집 공고」 클릭 → `/create` 진입

---

### 0:45 — 편집 & 미리보기 (50초)

> 「배너·문구만 바꿔도 되고, 레이아웃·글꼴·색은 디자인 패널에서 조정합니다. 모바일·PC 미리보기로 바로 확인해요.」

**동작**:
1. 제목 1줄 수정 (눈에 띄게)
2. **레이아웃** 1종 변경 (예: hero-overlay)
3. **디자인** → 글꼴 또는 강조 색 변경
4. **MO / PC** 토글

**한 줄**: 「코드 없이 폼만으로 랜딩이 완성됩니다.」

---

### 1:35 — 배포 & 로그인 (40초)

> 「배포는 로그인 후에만 가능하게 해서, 페이지가 대시보드에 저장되도록 했습니다. 비로그인이면 알럿 대신 **부드러운 모달**로 로그인을 유도합니다.」

**동작** (로그인 상태):
1. **배포하기** 클릭
2. `/p/{id}` 로 이동 → 공개 페이지 확인

**동작** (선택 — 비로그인):
1. 모달 노출 → 「계속 편집하기」로 닫기만 보여도 OK

---

### 2:15 — 공유 & 트래킹 (45초)

> 「마케터 시나리오에서는 채널별 UTM 링크가 중요해서, 트래킹 링크를 만들고 대시보드에서 방문을 봅니다.」

**동작**:
1. 공개 페이지에서 트래킹 링크 생성 (소유자일 때) 또는 대시보드로 이동
2. `/dashboard` → 페이지 카드 **⋮** → **방문 로그**
3. 유입 source/medium 차트 짧게 스크롤

---

### 3:00 — 마무리 (30초)

> 「MVP는 **만들기→배포→트래킹**까지 검증했고, 결제·커스텀 도메인·팀 기능은 PRD와 로드맵에 Phase 2로 설계해 두었습니다. 기획서와 동작하는 프로토타입을 함께 보시면 됩니다.」

**화면**: `docs/portfolio/README.md` 또는 Notion 링크 (있을 경우)

---
