Problem

왜 이 제품인가

이벤트·모집·세일·공지 페이지는 수명이 짧고 반복 제작이 많다. 그런데 실무에서는 —

외주·개발 요청

며칠~수주 소요.
긴급 이벤트에 맞지 않음.

미리캔버스·피그마

디자인은 되지만
배포·트래킹·수정이 분리됨.

노션·구글사이트

빠르지만
랜딩·전환(CTA)에 최적화되지 않음.

→ 빨리 만들고, 바로 공유하고, 효과를 보고, 다음에 또 빠르게 할 도구가 필요하다.
Solution

솔루션 & 포지셔닝

「이벤트·모집 랜딩을 10분에 배포」에만 집중한 초경량 빌더

핵심 가치

속도 — 템플릿 선택 후 폼 편집만으로 완성

배포 — 클릭 한 번으로 /p/{id} 공개 URL

트래킹 — UTM 링크 생성 + 방문 분석 대시보드

2×2 포지셔닝

Decision

주요 의사결정 (ADR)

결정대안선택 이유
JSON + 단일 렌더러페이지마다 커스텀 HTML유지보수 0, 템플릿 확장 용이
배포 시 로그인 필수익명 배포 허용페이지 소유·대시보드 일관성
트래킹을 대시보드로공개 페이지에 로그 노출운영자 UX·프라이버시
글꼴 드롭다운전체 그리드 노출편집 패널 공간·인지 부하 감소
로그인 모달alert()전환율·브랜드 톤 유지
코드는 Golden Path가 매끄럽게 동작하는 수준에서 의도적으로 중단했습니다. 범위 관리(scope discipline)가 포트폴리오의 핵심 메시지입니다.
